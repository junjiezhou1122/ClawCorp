# Plan: Implement Spec 002 — Task Board

## Context

The Chairman wants a simple task board (Backlog / Todo / In Progress / Review / Done) separate from the mission pipeline. Tasks are lightweight — just a title and status. The Review column has a "Push Back" action that returns tasks to In Progress with required written feedback. This is the Chairman's personal task management, independent of agents and missions.

## Implementation (4 commits)

### Commit 1: Server — Task CRUD route

**File: `server/src/routes/tasks.ts`** (new)

Follows the exact pattern of `missions.ts`:

- `TASKS_DIR = join(import.meta.dir, '../../../tasks')`
- `GET /` — list all tasks (readdir → read each state.json)
- `POST /` — create task: `{ id: "T-{timestamp}", title, status: "backlog", created_at, feedback: [] }`
- `PATCH /:id` — update status (drag-and-drop) or push back (append feedback)
- `DELETE /:id` — delete task directory

Broadcast events: `task:created`, `task:updated`, `task:deleted`

**File: `server/src/index.ts`** — add `app.route('/api/tasks', taskRoutes)`
