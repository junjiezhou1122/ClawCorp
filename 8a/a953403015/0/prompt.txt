# Plan: Implement Spec 003 — Smart Hire

## Context

The current "Hire Agent" flow is a manual form (ID, title, department, description, system_prompt). Spec 003 wants a **natural-language-in, hired-agent-out** flow: Chairman describes a need → system generates 3 candidates → each runs a standardized test in parallel → LLM scores results → best candidate auto-hired. All events stream to Live Log via WebSocket.

## Architecture

The heavy lifting (LLM candidate generation + LLM scoring) happens server-side. The client just provides a text input + shows progress via Live Log (already wired). We use Claude API (via `@anthropic-ai/sdk`) directly for the meta-prompts (candidate generation + scoring), NOT by spawning `claude` CLI.

## Implementation (3 commits)

### Commit 1: Server — Smart Hire engine + route

**File: `server/src/lib/SmartHire.ts`** (new)

Core engine with 3 phases:

1. **Generate candidates** — Call Claude API with the spec's meta-prompt to produce 3 candidate profiles (JSON array with id, title, department, description, reports_to, system_prompt)
2. **Run interviews** — For each candidate, spawn `claude --dangerously-skip-permissions -p "{test_task}"` in parallel with 60s timeout. Capture stdout as test output. Use standardized test tasks per department (from spec).
3. **Score & select** — Call Claude API with a scoring prompt: pass each candidate's profile + test output, ask for 0-10 scores on 4 dimensions (task_completion 40%, output_quality 30%, autonomy 20%, conciseness 10%). Parse scores, compute weighted average, pick winner.
4. **Hire winner** — Create agent directory + profile.json + memory.md (reuse existing `POST /api/hire` logic inline).
5. **Archive results** — Save full interview record to `archive/interviews/H-{timestamp}.json`.

All phases broadcast events: `hire:start`, `hire:candidates`, `hire:interview_start`, `hire:interview_done`, `hire:scores`, `hire:complete`, `hire:error`.

**File: `server/src/routes/hire.ts`** (edit)

Add `POST /api/hire/smart` route:
- Body: `{ description: string }`
- Validates non-empty description
- Calls `runSmartHire(description)` (async, returns immediately)
- Returns `{ ok: true, hireId }`

## Key decisions

- **Claude API via SDK**, not CLI spawning, for candidate generation + scoring (faster, structured output, no process overhead)
- **CLI spawning for interviews** — reuse the same `claude --dangerously-skip-permissions -p` pattern that AgentRunner uses, but simplified (no MCP injection, just a raw test in /tmp)
- **60s timeout** per interview candidate, all 3 run in parallel
- **No "review before hire" in v1** — auto-hire the winner (P2 feature per spec)
- **Archive interviews** to `archive/interviews/` for later inspection
