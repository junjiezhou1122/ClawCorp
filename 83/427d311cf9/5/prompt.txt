# Plan: Task Detail View — Per-Task Progress Dashboard

## Context

全局 Live Log 侧边栏把所有 agent 输出混在一起，dispatch 后 Chairman 看不到某个具体任务的进度。任务卡片只有标题/状态/按钮，没有详情。

**目标**: 点击任务卡片 → 全屏详情覆盖层，展示该任务的一切：过滤后的 live log、agent 树、产出物、通讯、反馈。移除全局 Live Log 侧边栏。

---

## Commit 1: Server — `GET /api/tasks/:id/detail`

**File: `server/src/routes/tasks.ts`**

新增常量 `MISSIONS_DIR`, `AGENTS_DIR`，加 4 个 helper 函数 + 1 个路由：

**Helpers:**
- `buildMissionTree(rootMissionId)` — BFS 扫描所有 `missions/*/state.json`，沿 `parent_mission` 链收集完整 mission 树，返回 ID 数组
- `scanArtifacts(missionId)` — readdir `missions/<id>/artifacts/`，返回相对路径
- `collectMessages(missionId)` — 读取 `missions/<id>/messages/*.json`
- `loadAgentProfile(agentId)` — 读取 `agents/<id>/profile.json`

**Route `GET /:id/detail`:**
1. 读 task state，无 `mission_id` 返回空数组
2. BFS 构建 mission 树
3. 加载树中所有 mission state
4. 收集 assignee → 加载 agent profiles
5. 每个 mission 扫 artifacts + messages
6. 返回 `{ task, missions, agents: Record<string,Agent>, artifacts: Record<string,string[]>, messages }`

## Commit 2: Client — Task Detail Overlay + 移除全局侧边栏

**File: `client/src/App.tsx`**

### 新增 state

```
selectedTask: Task | null
taskDetail: TaskDetail | null     // { task, missions, agents, artifacts, messages }
taskDetailLoading: boolean
taskMissionIds: Set<string>       // 用于 log 过滤的 mission ID 集合
logFilterAgent: string | null     // 按 agent 过滤 log
detailLogScrollRef               // 详情页 log 滚动 ref
selectedTaskRef                  // ref 同步 selectedTask，WS 回调用
```

### 任务卡片改造

- `<article>` 加 `onClick` → `openTaskDetail(t)`，用 `closest('button')` 防止按钮点击误触
- `cursor-grab` → `cursor-pointer`

### `openTaskDetail(task)` 函数

设 `selectedTask`，fetch `GET /api/tasks/:id/detail`，从 response 中 seed `taskMissionIds`

### WS handler 修改

- **`task:updated`**: 同步更新 `selectedTask`；若 `mission_id` 首次出现，自动 re-fetch detail
- **`mission:updated`**: `setMissions` 改为 "有则更新，无则新增"（当前只做更新）；若新 mission 的 `parent_mission` 在 `taskMissionIds` 中，扩展集合并加入 `taskDetail.missions`
- **`agent:start`**: 若 `missionId` 前缀匹配 `taskMissionIds` 中的任何 ID，扩展集合（利用 delegate 命名规则 `parentId-agentId-ts`）

### ESC 关闭

`useEffect` 监听 `keydown`，ESC 清空 `selectedTask`/`taskDetail`/`taskMissionIds`

### 详情覆盖层 JSX（全屏，非 modal-card）

```
┌─ Header ──────────────────────────────────────────────┐
│ ← Back  │  title + status badge + assigned_to pill    │
│         │  Dispatch / Push Back / Delete buttons       │
├─ Left Sidebar (w-72) ─┬─ Main: Live Log ──────────────┤
│ Routing reasoning      │ Agent filter chips + clear    │
│ Agent Tree (live dots) │ Filtered log lines            │
│ Artifacts list         │ (by taskMissionIds + agent)   │
│ Communications         │ Empty state if not dispatched │
│ Feedback history       │                               │
└────────────────────────┴───────────────────────────────┘
```

### Log 过滤逻辑

显示 `logs` 中满足以下条件的行：
- `missionId` 在 `taskMissionIds` 中，或前缀匹配集合中的 ID
- 或 `agentId === 'dispatch'`（routing 阶段的 log）
- 再叠加 `logFilterAgent` 过滤

### 移除全局侧边栏

- 删除整个 `<aside>` 块（Live Log 右栏，约 25 行）
- 父容器 `xl:flex-row` → `flex-col`（不再需要双栏）
- 删除 `logScrollRef` 和其 auto-scroll useEffect
- **保留** `logs` state 和 `appendLog` — 详情页 log 仍依赖它们

## 关键注意点

- **delegate 不广播**: sub-mission 只能通过 `agent:start` 事件发现，用 missionId 前缀匹配捕获
- **WS 闭包过期**: 用 `selectedTaskRef` 解决 WS 回调中 `selectedTask` 可能过期的问题
- **全屏 vs modal**: 详情用 `fixed inset-0` 全屏，不用 `modal-card max-w-xl`
